## 什么是1-bit大模型?

简单来说，**1-bit LLMs**（1比特大语言模型）是当前大模型架构的一种分支演进。

它通过将模型权重压缩到极低精度，试图在保持模型智能的同时，极大地降低计算成本和内存占用。

这一概念主要由微软研究院在关于 **BitNet b1.58** 的研究中推向大众视野。

### 1.核心原理

在传统的大模型中，模型权重通常使用 **FP16**（16位浮点数）。这意味着每个参数需要占用 16 位的存储空间。

**1-bit LLM 的做法则是：**

- **三进制权重：** 每个参数不再是复杂的浮点数，而是被限制在 *-1, 0, 1* 这三个值中。

- **计算简化：** 因为权重只有 -1、0 和 1，模型在推理时不再需要昂贵的**浮点乘法**，而只需要简单的**整数加减法**。


### 2. 优势

- **极速推理：** 整数加法比浮点乘法快得多，且能效比极高。依赖加法器即可，不需要复杂的张量计算电路。
    
- **内存暴减：** 相比于 FP16 模型，1-bit 模型的内存占用可以减少 **10 到 16 倍**。这意味着你可以在手机甚至智能手表上本地运行原本需要昂贵 GPU 才能跑动的大模型。1-bit 模型将参数从 16 位压缩到 1.58 位，意味着相同的内存带宽下，每秒可以传输 **10 倍以上** 的参数
  
- **突破壁垒：** 解决了目前 AI 领域最大的瓶颈——**内存带宽（Memory Wall）**。数据在内存和处理器之间传输的成本，往往比计算本身还要高。因为权重极小，原本需要放在显存里的数据，现在可以放进 CPU 的 **L3 缓存** 甚至是寄存器里。这让计算延迟从毫秒级降到了微秒级。
    
    
### 为什么是1.58b



- **数学表示：** $\log_2(3) \approx 1.58$。这就是为什么微软将其命名为 **BitNet b1.58**。

- **0 的重要性：** 在神经网络中，权重为 $0$ 相当于**稀疏化（Sparsity）**。这让模型能够有选择地“关闭”某些不重要的连接，这极大地增强了模型的特征表达能力，使其在极低精度下依然能保持智力。


### 如何训练？
你可能会问：如果权重只能是 -1, 0, 1，怎么用梯度下降来更新模型？（毕竟梯度通常是很小的浮点数）。

1-bit LLM 采用的是 **全精度潜变量（Full-precision Latent Weights）** 训练法：

1. **保存一份影子权重：** 在训练内存中保留一份 FP16 的高精度权重。
    
2. **前向传播用 1-bit：** 计算时将 FP16 量化为 $\{-1, 0, 1\}$。
    
3. **反向传播更新 FP16：** 计算出的微小梯度更新在 FP16 的“影子”上。
    
4. **收敛：** 随着训练进行，FP16 权重逐渐偏向某个整数，最终固化为 1-bit 模型。
    


## 如何在本地跑一个1-bit大模型?

> 系统 MacOs M4-24g

### 克隆官方库

```bash
# 要加 recursive  依赖submodule
git clone --recursive https://github.com/microsoft/BitNet.git && cd BitNet
# 安装依赖
python3 -m venv venv
pip install -r requirements.txt
```

### 手动下载模型

```bash
mkdir -p models/BitNet-b1.58-2B-4T
curl -L https://huggingface.co/microsoft/BitNet-b1.58-2B-4T-gguf/resolve/main/ggml-model-i2_s.gguf -o models/BitNet-b1.58-2B-4T/ggml-model-i2_s.gguf

## 下载完成后应该有1.2G左右的大小
```

### 编译启动

```bash
cmake -B build -DGGML_METAL=ON
cmake --build build --config Release

./build/bin/llama-cli \
  -m models/BitNet-b1.58-2B-4T/ggml-model-i2_s.gguf \
  -p "<|system|>\n你是一个有用的助手。<|end|>\n<|user|>\n你好，请用中文写一首关于春天的简短小诗。<|end|>\n<|assistant|>\n" \
  --color \
  -n 128
```

简单的推理测试：

```bash
./build/bin/llama-cli \
  -m models/BitNet-b1.58-2B-4T/ggml-model-i2_s.gguf \
  -p "<|system|>\n你是一个有用的助手。<|end|>\n<|user|>\n计算 12*1024等于多少<|end|>\n<|assistant|>\n" \
  --color \
  -n 128
# 输出 
# 12 * 1024 = 12 * (1000 + 32)
# = (12 * 1000) + (12 * 32)
# = 12000 + 384
# = 12384<|end|>
```
可能是参数太小，拆分的时候没有算对加法 手动提醒一下就能算出正确结果了

```bash
 ./build/bin/llama-cli \
  -m models/BitNet-b1.58-2B-4T/ggml-model-i2_s.gguf \
  -p "<|system|>\n你是一个有用的助手。<|end|>\n<|user|>\n计算 12*1024等于 12*(1000+24) 计算最终的结果等于多少<|end|>\n<|assistant|>\n" \
  --color \
  -n 128
# 输出
# 12 * 1024 = 12 * (1000 + 24)
# = 12 * 1000 + 12 * 24
# = 12000 + 288
# = 12288
# So, 12 * 1024 equals 12288.
```

## 个人看法

### 内脑模型的重现

从生物电信号的角度来看，1-bit大模型确实比传统的浮点数（FP16/32）模型更接近生物大脑的底层逻辑。


#### 1. 神经元的离散信号

生物神经元产生信号的基本单位是**动作电位（Action Potential）**，也就是俗称的“电脉冲”。

- **生物逻辑：** 神经元要么发放脉冲（1），要么不发放（0）。它不会发出一个“0.573”精度的电压信号。
    
- **1-bit 逻辑：** 1-bit（尤其是 BitNet 的 $\{-1, 0, 1\}$ 三进制）本质上模拟了这种**离散的状态切换**。
    
- **对比：** 传统深度学习追求精细的浮点数（如 0.000123），这在生物学上是极难实现的，因为环境噪声会轻易淹没这种微小的精度差异。

大量神经元置零之后不参与预算本身就是一个ZoomIn的过程。


#### 2. 能量效率

- **大脑的节能秘诀：** 大脑处理信息极少进行复杂的“乘法运算”。神经元接收到多个脉冲后，在细胞膜上进行的是**空间总和（Spatial Summation）**——本质上就是**加法**。
    
- **1-bit 的优势：** 在 BitNet 架构中，由于权重只有 $\pm1$ 或 $0$，矩阵运算中的“乘累加（MAC）”变成了单纯的“累加（Addition）”。
    
- **底层意义：** 乘法器是芯片上的能耗大户。去掉乘法，意味着 AI 芯片可以像大脑一样，用极低的功耗（约 20 瓦）处理极其复杂的逻辑。
    
#### 3. 容错性

- **生物系统：** 你的大脑每天都有成千上万的神经元死亡，或者受到生化环境波动的干扰，但你的意识依然连续。这是因为生物系统依赖的是**群体编码**而非单个数值的精确度。
    
- **1-bit 系统：** 低精度模型天然具有更好的**正则化效果**。它迫使模型学习更强健的特征，而不是死记硬背训练集里的微小噪声。这种“模糊的正确”比“精确的错误”更符合生命演化的逻辑。


### 现存的问题

- 基建的支持度还比较落后
- 相比浮点模型，概率切换非常陡峭 模型难训练 且很容易出现刻板行为
- 可能需要更复杂的结构和反馈机制来弥补精度上的不足
